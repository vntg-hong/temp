# 역할 (Role)
너는 React와 TypeScript에 능숙한 Senior Frontend Developer야. 
'여행/모임 정산(Dutch Pay)' 기능을 추가하려고 해. 
코드의 가독성과 상태 관리의 효율성을 최우선으로 고려해줘.
목적: 여행 총무가 혼자서 모든 지출을 기록하고, 환율이 적용된 정산 결과를 도출하여 친구들에게 공유하는 웹 앱.

# 기능 요구사항 (Requirements)
1. **멤버 관리**:
   - 정산에 참여할 친구들의 이름을 등록하고 삭제할 수 있어야 함.
2. **비용 기록 (지출 내역)**:
   - 지출 항목명, 금액, 결제자(누가 냈는지), 참여자(누가 같이 썼는지)를 입력받음.
3. **회비 관리**:
   - 초기 공동 회비를 설정하고, 현재 남은 잔액을 실시간으로 계산함.
4. **정산 결과 계산**:
   - 지출 내역을 바탕으로 '누가 누구에게 얼마를 보내야 하는지' 최종 정산 데이터를 산출함.
   - 예: (총 지출 / 인원수) - 본인이 낸 금액 = 정산 금액
📤 4단계: 공유 및 데이터 관리
결과 요약 생성: "누가 누구에게 얼마를 보내야 하는지" 최소 송금 경로 알고리즘 적용.


카톡 공유용 텍스트: 이모지를 활용한 깔끔한 요약 템플릿 생성 (클립보드 복사).

데이터 백업: 현재까지의 모든 내역을 JSON 파일로 내보내기 및 불러오기.
6. 차등 정산
7. 통화별 정산
   

앱을 새로고침해도 친구 목록과 지출 내역이 유지되게 해줘.
useEffect를 사용해서 상태가 변할 때마다 로컬 스토리지에 자동 저장되는 로직을 짜줘.
모든 데이터를 JSON 파일로 내보내고(Export) 가져오는(Import) 기능도 넣어줘.


# UI/UX 가이드
- 사용자가 직관적으로 내역을 입력할 수 있도록 폼(Form) 구조를 설계해줘.
- 리스트 형태로 지출 내역을 보여주고, 각 내역 옆에 '삭제' 버튼을 배치해줘.
- 상단에는 [총 예산 | 총 지출 | 남은 회비]가 한눈에 보이게 대시보드를 만들어줘.
